RevOps Assistant

tu copiloto de RevOps para HubSpot.
Cloud District · Edición 1 · Junio 2026

Introducción qué es y para quién.

RevOps Assistant es una aplicación de escritorio que ayuda a equipos de RevOps a documentar y gobernar la configuración de un portal de HubSpot: el mapa de propiedades y sus orígenes de datos, los objetos custom y los formularios. Cada proyecto agrupa el trabajo de un cliente o portal.

El manual está pensado para usuarios de negocio. Cada tarea se explica con sus prerrequisitos, los pasos numerados en orden, una representación de la pantalla y el resultado que debes esperar.

Principio clave

La aplicación nunca escribe en HubSpot de forma automática. Todo cambio sobre el portal se presenta como un cambio pendiente que revisas y aplicas de forma explícita, primero en Sandbox y después en Producción.

01

Primeros pasos para empezar.

¿Cómo abro la app y me oriento?
Antes de conectar nada conviene conocer la estructura de la aplicación: cómo se crean y abren los proyectos y cómo se reparte la pantalla.

Crear o abrir un proyecto

Prerrequisitos: Tener la aplicación instalada.  ·  Tiempo: 1 minuto

Al arrancar verás la pantalla de bienvenida con la lista de tus proyectos. Cada proyecto guarda la configuración de un portal.

  1. Abre la aplicación. Se muestra la pantalla de bienvenida.
  2. Para retomar un proyecto, pulsa sobre su tarjeta en la lista.
  3. Para empezar de cero, pulsa Nuevo proyecto y dale un nombre.
  4. Para traer un proyecto de otro equipo, pulsa Importar proyecto (ver el capítulo Guardar y compartir proyectos).
Resultado esperado

El proyecto queda abierto y se muestra el Dashboard, la pantalla de inicio de cada proyecto.

Conocer la interfaz

Prerrequisitos: Tener un proyecto abierto.  ·  Tiempo: 2 minutos

La ventana se organiza en tres zonas estables que encontrarás en casi todas las pantallas.

  1. Menú lateral (izquierda): acceso a Dashboard, CRM (Propiedades, Objetos custom, Formularios), Configuración y Ayuda.
  2. Barra superior: el título de la pantalla y la etiqueta del entorno de HubSpot activo (PROD o SANDBOX), visible siempre.
  3. Área de contenido: la pantalla en la que trabajas. Las acciones principales viven en una barra de acciones en su parte superior.
  4. En Ayuda tienes estos mismos tutoriales dentro de la app, en el idioma de la interfaz.
Resultado esperado

Sabes dónde está cada cosa y qué entorno de HubSpot está activo en cada momento.

¿En qué idioma está la app? Puedes elegir entre castellano, catalán, euskera, inglés, gallego, portugués y francés en Configuración; la preferencia se guarda.
02

Conectores HubSpot, Drive y MCP.

¿Con qué se conecta la app?
Los conectores son el puente entre la aplicación y tus herramientas. HubSpot es el portal que documentas y gobiernas; Google Drive aloja las hojas y documentos del proyecto; el servidor MCP permite que un cliente de IA opere sobre el proyecto activo.

Crear una Private App en HubSpot y obtener el token

Prerrequisitos: Ser administrador (Super Admin) del portal de HubSpot.  ·  Tiempo: 5 minutos

La aplicación se conecta mediante un Private App Token (PAT), el método recomendado por HubSpot para integraciones internas.

  1. Inicia sesión en HubSpot con una cuenta de administrador.
  2. Ve a Configuración (icono del engranaje, arriba a la derecha).
  3. En el menú lateral, abre Integraciones y luego Aplicaciones privadas.
  4. Pulsa Crear una aplicación privada.
  5. En Información básica escribe un nombre (por ejemplo, RevOps Assistant) y una descripción.
  6. En la pestaña Ámbitos (scopes) activa los permisos que necesites (ver la tabla de scopes).
  7. Pulsa Crear aplicación y confirma.
  8. Pulsa Mostrar token y Copiar. El token empieza por pat-.
Conjunto de scopes recomendado (habilita toda la app)
ScopeUsado por
crm.objects.contacts.readConexión básica
crm.schemas.contacts.read / .writePropiedades, Formularios
crm.schemas.deals.read / .writePropiedades
crm.schemas.companies.read / .writePropiedades
crm.schemas.custom.read / .writeObjetos custom

communication_preferences.readFormularios (consentimiento legal)
Importante
Anota qué scopes activas: la app no puede leer ni mostrar los scopes de un PAT (HubSpot no los expone vía API). Si falta uno, lo sabrás por un error 403 al usar la función concreta.
Resultado esperado

Tienes copiado un token tipo pat-eu1-xxxxxxxx, listo para introducirlo en la app.

¿Puedo cambiar los scopes después? Sí. Vuelve a la aplicación privada, ajusta los ámbitos y guarda; el token no cambia.
Tengo una cuenta sandbox. Repite los pasos dentro del portal sandbox para obtener un token independiente, que usarás en el entorno Sandbox de la app.

Conectar la app con HubSpot

Prerrequisitos: Tener un token PAT y un proyecto abierto.  ·  Tiempo: 2 minutos

Con el token en el portapapeles, la conexión se configura en la pantalla del conector.

  1. Abre tu proyecto y, en el menú lateral, pulsa Configuración.
  2. En Conectores, pulsa HubSpot.
  3. Selecciona la pestaña del entorno a configurar: Producción o Sandbox.
  4. Pega el token en el campo Private App Token.
  5. Pulsa Guardar. La app valida el token contra HubSpot.
Resultado esperado

Un indicador Conectado (badge lima), la línea Portal: nombre (id) y la versión de API en uso. Si el token no es válido, verás el motivo y no se guardará.

¿Se ve mi token en algún sitio? No. El campo es de tipo contraseña, el token se guarda cifrado en el llavero del sistema y se oculta en cualquier registro.
¿Cómo desconecto el portal? En la misma pantalla, con el entorno seleccionado, pulsa Revocar.

Configurar y cambiar entre Producción y Sandbox

Prerrequisitos: Tener el conector HubSpot configurado en al menos un entorno.  ·  Tiempo: 3 minutos

Cada proyecto admite dos entornos independientes, cada uno con su token y portal. El entorno activo es el destino de toda escritura.

  1. Abre Configuración, Conectores, HubSpot.
  2. Para el sandbox, selecciona la pestaña Sandbox, pega su token y pulsa Guardar.
  3. Para cambiar el entorno activo, selecciona la pestaña deseada.
  4. Si está conectado y no es el activo, pulsa Usar como entorno activo.
Resultado esperado

La etiqueta de la barra superior (PROD o SANDBOX) refleja el entorno activo. Las lecturas pueden ir a cualquier entorno; las escrituras usan siempre el activo.

¿Para qué sirve el sandbox? Para probar cambios sin tocar datos reales. Configura primero en sandbox, valida y replica en producción.
¿Puedo tener solo producción? Sí. El sandbox es opcional.

Conectar Google Drive

Prerrequisitos: Un proyecto abierto y una cuenta de Google. Tener a mano el ID de cliente de OAuth de Google Cloud.  ·  Tiempo: 3 minutos

Drive aloja las hojas y documentos que la app genera del proyecto. La conexión usa OAuth; el acceso queda acotado a los archivos que la app crea o que tú selecciones.

  1. Abre Configuración, Conectores, Google Drive.
  2. En Credenciales de Google Cloud, pega tu ID de cliente (termina en .apps.googleusercontent.com) y, si tu cliente lo exige, el secreto. Pulsa Guardar.
  3. Pulsa Conectar con Google. Se abre el navegador del sistema con la pantalla de autorización.
  4. Elige la cuenta, revisa los permisos y acepta.
  5. Vuelve a la app: verás el estado Conectado junto a tu correo.
Resultado esperado

La pantalla muestra Conectado como tu-correo y aparece la sección Carpeta de trabajo.

¿La app ve todos mis archivos de Drive? No. El permiso (drive.file) limita el acceso a los archivos que crea la app o que elijas explícitamente.
¿Por qué se abre el navegador? Por seguridad: te autenticas donde ya tienes sesión y la app nunca ve tu contraseña.

Seleccionar la carpeta de trabajo y sincronizar

Prerrequisitos: Haber conectado la cuenta de Google.  ·  Tiempo: 2 minutos

La carpeta de trabajo es donde la app guarda los documentos gestionados del proyecto.

  1. En Google Drive, localiza Carpeta de trabajo y pulsa Seleccionar carpeta.
  2. Navega tu Drive desde Mi unidad y entra en la carpeta deseada.
  3. Pulsa Seleccionar esta carpeta.
  4. En la sección Sincronización, pulsa Sincronizar para traer el estado actual.
  5. Revisa los Archivos gestionados: cada uno muestra Sincronizado, En conflicto o Pendiente.
Resultado esperado

La carpeta queda fijada y la fecha de última sincronización se actualiza. Sin conflictos, todo aparece como Sincronizado.

¿Qué versión manda si hay diferencias? Google Drive es la fuente de verdad; si detecta cambios locales más recientes, marca el archivo En conflicto para que decidas.
¿Veo unidades compartidas? Esta versión del selector navega solo Mi unidad.

Conectar un cliente MCP

Prerrequisitos: Un proyecto abierto. Node.js/npx instalado para el puente local.  ·  Tiempo: 3 minutos

El servidor MCP permite que un cliente de IA (por ejemplo, Claude Desktop) opere sobre el proyecto activo mediante un conjunto de tools.

  1. Abre Configuración y, en conectores, pulsa API / MCP.
  2. Activa el interruptor Servidor MCP. El estado pasa a activo y se muestra el Puerto (por defecto 3741).
  3. Pulsa Copiar configuración para obtener el snippet de tu cliente.
  4. Pega el snippet en la configuración de tu cliente MCP y reinícialo.
Importante
El servidor escucha solo en 127.0.0.1 (tu equipo); no es accesible desde la red. El token viaja en la cabecera x-api-key; al regenerarlo, el anterior deja de funcionar de inmediato.
Resultado esperado

El cliente se conecta y lista las tools disponibles (al menos mcp_health, que confirma la conexión y el proyecto activo).

¿Tengo que dejar la app abierta? Sí. El servidor corre dentro de la app; si la cierras, el cliente pierde la conexión.
03

Propiedades el mapa de datos.

¿Cómo documento y gobierno las propiedades?
El mapa de propiedades es el listado maestro de las propiedades del portal y su definición prevista. Documenta de dónde vienen los datos (orígenes), cómo se transforman, y gestiona los cambios que hay que aplicar en HubSpot.

Añadir una propiedad al mapa

Prerrequisitos: Un proyecto abierto. Conviene tener el conector HubSpot configurado.  ·  Tiempo: 5 minutos

Puedes incorporar propiedades importándolas al sincronizar, o añadiéndolas manualmente cuando aún no existen en el portal.

  1. Entra en CRM, Propiedades.
  2. Pulsa Propiedad en la barra de acciones. Se abre el diálogo Añadir propiedad.
  3. Rellena Nombre técnico (HubSpot), Etiqueta, Objeto, Tipo, Tipo de campo, Grupo y, opcionalmente, Descripción.
  4. Pulsa Crear. La propiedad aparece en la tabla con estado missing.
  5. Haz clic en la fila para abrir el panel lateral y asociarle orígenes.
Importante
Crear la propiedad aquí NO la crea en HubSpot. Eso requiere aplicar el cambio pendiente de forma explícita.
Resultado esperado

La propiedad queda en el mapa con estado missing. Al sincronizar se generará un cambio pendiente de tipo Crear propiedad.

¿Puedo editar una propiedad importada? Puedes editar su etiqueta y descripción; el resto refleja el estado real del portal.

Gestionar orígenes de datos

Prerrequisitos: Un proyecto abierto.  ·  Tiempo: 5 minutos

Un origen representa de dónde proceden los datos de una propiedad: una integración, una migración, la entrada manual por usuarios, o un workflow de HubSpot.

  1. Entra en CRM, Propiedades.
  2. Pulsa Orígenes (n) en la barra superior.
  3. Para crear uno, rellena Nombre, Tipo (Integración, Migración, Usuario o Workflow) y, opcionalmente, Descripción.
  4. Pulsa Añadir origen. Aparece en la lista al instante.
  5. Para eliminar un origen, usa el icono de papelera (se borran también sus mapeos).
Resultado esperado

Los orígenes quedan guardados en el proyecto y se reflejan en la hoja 01_Origenes del Google Sheets del mapa.

¿Qué pasa con las propiedades si elimino el origen? Se eliminan los mapeos de ese origen, pero las propiedades permanecen.

Mapear orígenes y transformaciones

Prerrequisitos: Al menos una propiedad en el mapa y un origen creado.  ·  Tiempo: 7 minutos

Un mapeo conecta una propiedad con un origen, indicando de qué campo procede y qué equivalencias de valor aplicar para que encajen en HubSpot.

  1. En CRM, Propiedades, haz clic en la propiedad. Se abre el panel lateral.
  2. En Orígenes mapeados, pulsa Añadir origen.
  3. En el diálogo, elige el Origen e indica el Campo origen (por ejemplo, Account_Tier__c).
  4. Pulsa Añadir regla por cada equivalencia de valor (por ejemplo, GOLD → enterprise).
  5. Pulsa Guardar. El mapeo aparece en el panel y en la columna Orígenes de la tabla.
Resultado esperado

El mapeo se guarda y se refleja en la hoja 03_Mapeo_Origenes del Sheets, listo para el contrato JSON del origen.

¿Puedo definir lógica compleja? No. Por seguridad solo se admiten equivalencias de valor, nunca scripts.
¿Una propiedad puede tener varios orígenes? Sí. Añade un mapeo por cada origen que la alimente.

Sincronizar el mapa con HubSpot

Prerrequisitos: Conector de HubSpot configurado.  ·  Tiempo: 3 minutos

La sincronización compara la definición del proyecto con el estado real del portal y clasifica cada propiedad. Solo lee; nunca modifica HubSpot.

  1. Entra en CRM, Propiedades.
  2. Comprueba en la barra superior qué entorno está activo.
  3. Pulsa Sincronizar HubSpot.
  4. Al terminar verás un resumen: al día, divergentes y sin crear.
Estados de propiedad
EstadoSignificado
exists (lima)Existe en HubSpot y coincide con la definición
divergent (gris)Existe pero difiere; genera cambios pendientes
missing (gris oscuro)No existe; genera un cambio pendiente de creación
Resultado esperado

Cada propiedad muestra su badge de estado. El mapa actualizado se vuelca al Google Sheets del proyecto.

¿La sincronización modifica HubSpot? No. Solo lee. Cualquier diferencia se propone como cambio pendiente.

Aplicar cambios en HubSpot

Prerrequisitos: Haber sincronizado y tener cambios pendientes.  ·  Tiempo: 5 minutos

Los cambios se presentan como una lista que revisas y aplicas de forma explícita. La recomendación es aplicar primero en Sandbox, validar, y después en Producción.

  1. Entra en CRM, Propiedades.
  2. Pulsa Cambios pendientes (n). Cada tarjeta muestra la propiedad, la operación y la llamada de API.
  3. Pulsa Aplicar en Sandbox y valida en tu sandbox de HubSpot.
  4. Pulsa Aplicar en Producción para ejecutarlo en el portal real.
  5. Si un cambio no procede, pulsa Descartar.
Importante
Aplicar en producción modifica el portal real de un cliente. Confirma el entorno activo antes de continuar.
Resultado esperado

Cada cambio refleja su estado por entorno (sandbox y producción). No se considera completado hasta aplicarse en producción.

¿Y si la llamada falla? El cambio no se marca como aplicado y verás el mensaje de error. Corrige y reintenta.

Exportar JSON por origen

Prerrequisitos: Al menos un origen con propiedades mapeadas.  ·  Tiempo: 2 minutos

La exportación genera un contrato de integración en JSON con las propiedades de un origen, su campo origen y las reglas de transformación.

  1. Entra en CRM, Propiedades.
  2. Pulsa Exportar JSON. Se despliega un menú con un elemento por origen.
  3. Selecciona el origen a exportar.
  4. El navegador descarga un fichero {nombre-origen}_{fecha}.json.
Resultado esperado

Un JSON con schema_version, datos del origen, fecha y las propiedades mapeadas con sus transformaciones.

¿Por qué por origen y no todo junto? Cada origen suele ser una integración distinta; el contrato por origen es justo lo que necesita quien la desarrolla.
04

Objetos custom tus entidades propias.

¿Cómo modelo lo que HubSpot no trae de serie?
Los objetos custom representan entidades propias de tu negocio (máquinas, contratos, vehículos) que no encajan en los objetos estándar. La app crea y gestiona la definición del objeto; los registros se gestionan después en HubSpot.

Crear un objeto custom

Prerrequisitos: Conector de HubSpot configurado y proyecto abierto.  ·  Tiempo: 5-10 minutos

El asistente define el objeto en cuatro bloques: identidad, propiedades iniciales, visualización y asociaciones.

  1. En CRM, abre Objetos custom y pulsa Objeto custom.
  2. Identidad: Nombre interno (no se podrá cambiar después), Etiqueta singular y plural, y Descripción.
  3. Propiedades iniciales: añade cada propiedad con Nombre interno, Etiqueta, Tipo y Tipo de campo; marca Único si identifica el registro.
  4. Visualización: elige Propiedad principal (obligatoria), Requeridas, Secundarias y Búsqueda.
  5. Asociaciones (opcional): con qué objetos podrá relacionarse.
  6. Pulsa Guardar. El objeto se añade como borrador con un cambio pendiente de tipo crear.
Resultado esperado

El objeto aparece como borrador. Todavía no existe en HubSpot: aplica su cambio pendiente en sandbox y luego en producción.

¿Por qué no puedo cambiar el nombre interno luego? Es una restricción de HubSpot: el nombre es inmutable una vez creado. Las etiquetas sí se cambian.
¿Tengo que crear todas las propiedades aquí? No. Puedes crear el objeto con las mínimas y añadir más después desde Propiedades.

Editar un objeto custom

Prerrequisitos: Al menos un objeto custom creado o en borrador.  ·  Tiempo: 3-5 minutos

Puedes ajustar etiquetas, propiedades de visualización, requeridas y asociaciones. El nombre interno no se puede cambiar.

  1. Abre CRM, Objetos custom y haz clic en el objeto.
  2. Pulsa Editar: se abre el asistente con los datos actuales.
  3. Cambia etiquetas, propiedad principal, requeridas, búsqueda o asociaciones (el nombre interno aparece bloqueado).
  4. Pulsa Guardar. Si la definición difiere de HubSpot, se genera un cambio pendiente de tipo actualizar schema.
Resultado esperado

Si hay diferencias, el objeto pasa a estado diverge y aparece un cambio pendiente. Aplícalo en sandbox y producción.

Quiero añadir una propiedad y marcarla requerida. Primero debe existir en HubSpot. Créala desde Propiedades y luego, en la edición, márcala como requerida.

Aplicar cambios de objetos en HubSpot

Prerrequisitos: Objetos custom con cambios pendientes.  ·  Tiempo: 3-5 minutos

Como en propiedades, los cambios se acumulan como pendientes y los aplicas tú, primero en sandbox y luego en producción.

  1. Abre CRM, Objetos custom y pulsa Cambios pendientes.
  2. Por cada cambio verás la operación (crear / actualizar schema / archivar) y su estado por entorno.
  3. Pulsa Aplicar en Sandbox y revisa el resultado.
  4. Cuando estés conforme, pulsa Aplicar en Producción.
Resultado esperado

El estado del cambio muestra sandbox OK y, después, producción OK. Al crear, la app guarda el identificador que HubSpot asigna en cada entorno (son distintos).

Me da error al actualizar o archivar. Asegúrate de que el objeto ya existe en ese entorno (debe haberse creado allí primero).

Archivar un objeto custom

Prerrequisitos: Un objeto custom creado en HubSpot.  ·  Tiempo: 3 minutos

Archivar elimina la definición del objeto en HubSpot. Es una acción destructiva y requiere doble confirmación.

  1. Abre el objeto en CRM, Objetos custom.
  2. En el panel de detalle pulsa Archivar y confirma la segunda vez (Confirmar archivado).
  3. Se genera un cambio pendiente de tipo archivar.
  4. Aplica el cambio en el entorno correspondiente.
Resultado esperado

El objeto pasa a archivado una vez aplicado. Si HubSpot lo rechaza, verás el motivo (normalmente quedan registros, asociaciones o propiedades).

HubSpot me da error al archivar. Solo permite archivar cuando se han eliminado antes todos sus registros, asociaciones y propiedades.
¿El borrado definitivo? No está disponible desde la app en esta versión; hazlo desde HubSpot si lo necesitas.
05

Formularios captura coherente.

¿Mis formularios recogen lo que deben?
La gestión de formularios importa los formularios del portal, permite crear formularios solo-campos a partir de un origen, asociarlos a orígenes y medir su cobertura para detectar campos que faltan.

Importar los formularios existentes

Prerrequisitos: Conector de HubSpot configurado (con el scope forms).  ·  Tiempo: 3 minutos

La importación trae a la app todos los formularios del portal para asociarlos a orígenes y revisar su cobertura. Solo lee.

  1. Entra en CRM, Formularios.
  2. Comprueba en la barra superior qué entorno está activo.
  3. Pulsa Sincronizar HubSpot.
  4. Al terminar verás cuántos formularios se han importado y actualizado.
Tipos de formulario
TipoQué es
hubspotFormulario de HubSpot (nuevo o legacy). Único tipo que la app puede crear
capturedFormulario HTML externo capturado (legacy)
flowFormulario emergente (pop-up)
blog_commentFormulario de comentarios de blog
Resultado esperado

La tabla muestra cada formulario con su tipo y cobertura. Puedes buscar por nombre y filtrar por tipo o cobertura.

¿La importación modifica HubSpot? No. Solo lee. El estado de verdad sigue siendo HubSpot.

Crear un formulario nuevo (solo campos)

Prerrequisitos: Al menos un origen y sus entradas de propiedades en CRM, Propiedades.  ·  Tiempo: 4 minutos

El asistente crea un formulario de HubSpot definiendo sus campos a partir de un origen. El diseño y la lógica se mantienen en HubSpot.

  1. Entra en CRM, Formularios y pulsa Formulario.
  2. Escribe el nombre y elige el objeto de HubSpot (estándar o custom).
  3. Marca uno o varios orígenes; la app preselecciona los campos que definen.
  4. Ajusta la lista: marca o desmarca campos y edita etiqueta, obligatorio y oculto.
  5. Pulsa Crear. Se genera un cambio pendiente de tipo crear formulario.
  6. Aplica el cambio desde Cambios pendientes.
Resultado esperado

Al aplicar, el formulario se crea en HubSpot con tipo hubspot y queda asociado a los orígenes elegidos.

¿Por qué solo campos? El alcance de la app es la estructura de campos y su relación con los orígenes; el diseño se mantiene en HubSpot.

Asociar un formulario a un origen

Prerrequisitos: Formularios importados y al menos un origen definido.  ·  Tiempo: 2 minutos

Asociar un formulario a uno o varios orígenes le dice a la app contra qué propiedades medir la cobertura.

  1. Entra en CRM, Formularios y haz clic en el formulario.
  2. Pulsa Asociar a origen.
  3. Elige el objeto contra el que se evalúa la cobertura.
  4. Marca uno o varios orígenes y pulsa Guardar.
Resultado esperado

El formulario muestra los orígenes asociados como etiquetas y el panel calcula la cobertura para cada uno.

¿Dónde se guardan las asociaciones? En el estado local del proyecto (no en HubSpot). Opcionalmente puedes volcarlas a Google Sheets.

Revisar la cobertura y añadir campos en bloque

Prerrequisitos: Un formulario asociado a uno o varios orígenes.  ·  Tiempo: 3 minutos

La cobertura indica si un formulario contiene todos los campos que su origen exige. Cuando faltan, la app los prepara todos de una vez como cambio pendiente.

  1. Entra en CRM, Formularios y observa el badge de cobertura de cada fila.
  2. Haz clic en un formulario para ver el detalle por origen: esperadas, presentes y faltantes.
  3. Si faltan, pulsa Añadir campos que faltan (N).
  4. Se genera un cambio pendiente que incluye solo los campos ausentes.
  5. Revisa y aplica el cambio desde Cambios pendientes.
Estados de cobertura
EstadoSignificado
completo (lima)Todos los campos del/los origen(es) están presentes
faltan NFaltan N campos definidos por el origen
sin origen (gris)No está asociado a ningún origen; no se puede evaluar
Resultado esperado

Cada propiedad esperada aparece como presente o ausente. No se duplican campos: solo se añaden los que faltan.

¿La cobertura mira los valores? No. Solo comprueba que el campo (la propiedad destino) exista en el formulario.

Sincronizar los cambios con HubSpot

Prerrequisitos: Al menos un cambio pendiente de formularios.  ·  Tiempo: 3 minutos

Ningún cambio se escribe automáticamente. Se acumulan como pendientes y los aplicas tú, probando antes en sandbox.

  1. Entra en CRM, Formularios y pulsa Cambios pendientes (N).
  2. Revisa cada cambio: resumen, operación y estado por entorno.
  3. Comprueba el entorno activo en la barra superior.
  4. Pulsa Aplicar en Sandbox y, cuando estés conforme, Aplicar en Producción.
Resultado esperado

Tras aplicar, el formulario se crea o actualiza en HubSpot en el entorno elegido. No se considera completado hasta aplicarse en producción.

¿Y si falta el scope forms? HubSpot devuelve un error 403 y la app te lo muestra; el cambio no se marca como aplicado.
¿Se pueden borrar formularios? No. El borrado de formularios queda fuera de alcance.
06

Estado del proyecto una mirada rápida.

¿Dónde tengo trabajo pendiente?
El Dashboard y la Vista general de CRM son pantallas de solo lectura que resumen el estado del proyecto y te llevan directo a donde hay trabajo.

Entender el Dashboard

Prerrequisitos: Un proyecto abierto.  ·  Tiempo: 2 minutos

El Dashboard es la primera pantalla al abrir un proyecto. Resume qué tienes conectado y qué cambios quedan por aplicar.

  1. Abre un proyecto desde la bienvenida. Verás el Dashboard.
  2. En Conectores revisa el estado de HubSpot, Google Drive y API / MCP.
  3. Pulsa Configurar en una tarjeta para ir a ese conector.
  4. En Cambios pendientes verás cuántos cambios sin aplicar hay en Propiedades, Objetos custom y Formularios.
  5. Pulsa Revisar para ir a esa área. Si pone Al día, no hay nada pendiente.
Resultado esperado

Sabes en segundos si tus conectores están listos y dónde tienes trabajo pendiente.

Solo veo Primeros pasos. Es normal si aún no has configurado ningún conector: sigue los enlaces para conectarlos.
Los contadores no cuadran. Usa el botón de recarga de la cabecera para volver a consultar el estado del portal.

Vista general de CRM

Prerrequisitos: Un proyecto abierto.  ·  Tiempo: 1 minuto

La pantalla CRM resume las tres áreas de trabajo en un solo vistazo.

  1. En el menú lateral, pulsa CRM.
  2. Cada tarjeta muestra el total de elementos y cuántos cambios pendientes hay (o Al día).
  3. Pulsa Abrir en una tarjeta para ir a esa área.
  4. Si HubSpot no está conectado, verás un aviso con enlace para configurarlo.
Resultado esperado

Sabes de un vistazo cuánto tienes en cada área del CRM y dónde hay trabajo pendiente.

¿En qué se diferencia del Dashboard? El Dashboard cubre todo el proyecto; esta vista se centra en el CRM y añade el total de elementos por área.
07

Guardar y compartir proyectos portables.

¿Cómo llevo un proyecto a otro equipo?
Un proyecto completo se guarda en un único archivo .rvproj para compartirlo o trasladarlo. El archivo no contiene credenciales: solo referencias y la configuración que gestiona la app.

Exportar un proyecto a un archivo

Prerrequisitos: Tener el proyecto en la lista de bienvenida.  ·  Tiempo: 1 minuto

La exportación empaqueta el mapa de propiedades, los objetos custom y los formularios.

  1. En la bienvenida, localiza el proyecto en la lista.
  2. Pulsa el icono Exportar (flecha hacia abajo) en su fila.
  3. Elige dónde guardar el archivo .rvproj y confirma.
Resultado esperado

Un archivo .rvproj listo para compartir. No incluye tokens ni contraseñas: solo referencias (portal, carpeta) y el estado de cada característica del CRM.

¿Incluye mis credenciales? No. Quien lo importe deberá reconectar sus propios conectores.

Importar un proyecto desde un archivo

Prerrequisitos: Un archivo .rvproj exportado previamente.  ·  Tiempo: 2 minutos

Recrea un proyecto a partir de un archivo, con un resumen previo y comprobación de integridad.

  1. En la bienvenida, pulsa Importar proyecto.
  2. Selecciona el archivo .rvproj y confirma.
  3. Revisa el resumen previo: nombre, conectores referenciados y contenido.
  4. Si ya existe un proyecto con el mismo identificador, elige Crear copia (por defecto) o Sobrescribir.
  5. Pulsa Importar y, después, reconecta HubSpot y Google Drive con tus credenciales.
Resultado esperado

El proyecto aparece en la lista y queda abierto, con su mapa de propiedades, objetos custom y formularios.

¿Y si una sección es de una versión más nueva? Se omite al importar pero se conserva en el archivo, de modo que no se pierde si vuelves a exportar.
¿Y si el archivo está alterado? Si la comprobación de integridad no cuadra, verás un aviso en el resumen antes de importar.

Glosario de terminos.

PAT (Private App Token)
Token de aplicación privada de HubSpot con el que la app se conecta. Empieza por pat-.
Scope
Permiso concreto que concede el token (por ejemplo, leer propiedades de contactos). HubSpot no permite consultarlos vía API.
Entorno (Producción / Sandbox)
Portal de HubSpot de destino. El entorno activo recibe todas las escrituras.
Origen
De dónde proceden los datos de una propiedad: integración, migración, usuario o workflow.
Cambio pendiente
Operación sobre HubSpot (crear, actualizar, archivar) preparada por la app que tú aplicas de forma explícita.
Cobertura
Medida de si un formulario contiene todos los campos que su origen exige para un objeto.
Estado missing / divergent / exists
Situación de una propiedad frente al portal: sin crear, distinta o coincidente.
.rvproj
Archivo portable que empaqueta un proyecto (sin credenciales) para compartirlo o trasladarlo.
MCP
Protocolo que permite a un cliente de IA operar sobre el proyecto activo mediante tools.
